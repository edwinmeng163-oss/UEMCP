## Description

Project-local Python user tool that dry-runs or places a controllable ground character in the current level for PIE. It prefers known ThirdPerson template character classes, never treats `BP_TwinStickCharacter` as a third-person success, and reports any basic `Character` or caller-supplied class as a fallback in `structuredContent`.

Inputs include `location`, `label`, `classPath`, `characterClassPolicy`, and `autoPossessPlayer`. Real writes use editor subsystems instead of deprecated `EditorLevelLibrary`, avoid setting a bare `GameModeBase`, mark the actor/map dirty, save dirty map packages, and report map plus OFPA external-actor package diagnostics.
