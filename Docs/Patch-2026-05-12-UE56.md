## 中文

### 改动

- **真实 build 验证**：在本地 UE 5.6.1 + UE 5.7.4 各跑了一次完整 build + 编辑器启动，全部通过。Audit 之前漏掉一个真实问题已经修了。
- **`Misc/StringOutputDevice.h` shim**：这个头文件是 UE 5.7 才有的。`UnrealMcpEditorTools.cpp` / `UnrealMcpWidgetTools.cpp` 现在用 `#if ENGINE_MINOR_VERSION >= 7` guard 包起来，5.6 上从 `Containers/UnrealString.h` 传递引入。
- **三层防御已加上**：
  1. `UnrealMcpModule.cpp` 编译时 `#error`：低于 5.6 直接编译失败，错误信息指向本文档
  2. `UE_LOG: UnrealMcp plugin built against UE x.y.z` 运行时启动日志
  3. `Tools/check_ue56_compat.py` 静态 linter，**理解 #if 上下文**——guard 内的 5.7-only include 不报警，guard 外才报
- **修了一个 `.uplugin` 陷阱**：`"EngineVersion": "5.6.0"` 字段被 UE 当成 **strict-match** 而不是最低版本，会让 5.7 上插件被 skip。已删除该字段。
- **host project Target.cs** 切到 `BuildSettingsVersion.Latest` + `EngineIncludeOrderVersion.Latest`，对 5.6 和 5.7 都正确。
- **`.uproject` `EngineAssociation`** 设为 5.6（向上兼容，5.7 用户右键 Switch Engine Version 即可）。

### 操作上需要做什么

**普通用户**：不需要做任何事。`git pull` 后 build 即可。

**用 Xcode 26+ 跑 UE 5.6 的 macOS 用户**：UE 5.6 的 `Engine/Config/Apple/Apple_SDK.json` 把 Xcode 上限锁在 16.9。需要本地把 `"MaxVersion": "16.9.0"` 改成 `"MaxVersion": "27.0.0"`（这是引擎安装目录的改动，不入仓库）。Xcode 16.x 的用户不受影响。

## English

### What changed

- **Real build verification**: A clean build + editor launch was run against both UE 5.6.1 and UE 5.7.4 locally. Both passed. One real incompatibility that the documentation-only audit had missed is now fixed.
- **`Misc/StringOutputDevice.h` shim**: that header was added in UE 5.7. `UnrealMcpEditorTools.cpp` / `UnrealMcpWidgetTools.cpp` now wrap the include in `#if ENGINE_MINOR_VERSION >= 7`; on 5.6 the `FStringOutputDevice` symbol comes in transitively via `Containers/UnrealString.h`.
- **Three-layer defense in place**:
  1. Compile-time `#error` in `UnrealMcpModule.cpp` if the engine is below 5.6 — fails fast with a pointer to this patch note.
  2. `UE_LOG: UnrealMcp plugin built against UE x.y.z` startup line so the targeted engine is visible in the editor log.
  3. `Tools/check_ue56_compat.py` static linter that **parses #if context** — a 5.7-only include inside a proper guard does not raise, an unguarded one does.
- **`.uplugin` trap fix**: `"EngineVersion": "5.6.0"` was being treated as a **strict-match** check, not a minimum, and caused UE 5.7 to skip loading the plugin. The field has been removed.
- **Host target files** now use `BuildSettingsVersion.Latest` + `EngineIncludeOrderVersion.Latest`, valid on both 5.6 and 5.7.
- **`.uproject` `EngineAssociation`** is now 5.6 (forward-compatible; 5.7 users get the project upgraded automatically by right-click → Switch Unreal Engine Version).

### What users need to do

**Most users**: nothing. `git pull` and rebuild.

**macOS users on Xcode 26+ targeting UE 5.6**: UE 5.6 pins the macOS Xcode upper bound at 16.9 in `Engine/Config/Apple/Apple_SDK.json`. Locally raise `"MaxVersion": "16.9.0"` to `"MaxVersion": "27.0.0"` to unblock the build. This is a per-machine engine-install patch and is not committed to the repo. Xcode 16.x users are unaffected.

### 変更内容

  1. `UnrealMcpModule.cpp` のコンパイル時 `#error`: 5.6 未満のエンジンで即座にビルド失敗し、エラーメッセージはこのパッチノートを示します。
